1.设置线程池的大小，主要考虑以下几个点：
     CPU数量
            内存
            任务类型：计算密集型、I/O密集型、或者两者皆可
            是否需要JDBC连接这样的稀缺资源
            如果执行的是不同类别的任务，并且行为相差很大，那么应该考虑使用多个线程池，从而使每个线程池可以根据各自的工作负载来调整
            
           针对计算密集型的任务，如果拥有N个cpu的系统上，当线程池的数量为N+1时，可以实现最优的利用率；
           针对含I/O操作或者其他阻塞操作的任务，由于线程不会一直执行，因此线程池规模应该更多；
 
          要正确的设置线程池大小，必须要估算出任务的等待时间和计算时间的比值，需要通过一些分析或监控工具来获得，此外还可以通过另一种方式来调节
线程池的大小：在某个基准负载下，分别设置不同大小的线程池来运行应用程序，来观察CPU利用率的水平
